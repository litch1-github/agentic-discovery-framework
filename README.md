# Agentic Discovery Framework

**Фреймворк для проведения Discovery в эпоху AI-агентов и трансформации PDLC.**
> SDD без Discovery = прежняя история про «самое подробное тех. задание». Без Discovery спецификация фиксирует «что построить», но не «зачем и в какой форме».

## Проблема
При внедрении AI-агентов команды часто скатываются к **Адаптации** (автоматизации старого процесса), вместо **Перепроектирования** (создания нового процесса вокруг возможностей агента). Это приводит к тому, что дорогая AI-технология просто ускоряет legacy-процессы, не давая нелинейного прироста.

## Что внутри
Этот репозиторий содержит готовый инструментарий для Product Engineers и системных аналитиков:
- **Методология**: Правила проведения Discovery, матрица применимости агента (5 критериев), работа с антипаттернами.
- **Шаблоны (Templates)**: Готовые Markdown-шаблоны для PR/FAQ, Outcome Hypothesis, Opportunity Statement.
- **Кейс (Examples)**: Полный разбор внедрения AI-агента в ипотечный скоринг (сокращение Time-to-Decision с 10 дней до 4 часов за счет перепроектирования).

## Матрица применимости агента
Как понять, какой уровень автоматизации нужен? (См. `templates/`)
1. Детерминированная логика
2. AI-Augmented
3. AI-Native
4. Agentic (Human-on-the-loop)

## Как использовать
1. Скопируйте нужный шаблон из папки `/templates`.
2. Проведите сессию с командой, используя правила фасилитации.
3. Задокументируйте решение (Адаптация vs Перепроектирование).
4. Сформируйте Outcome Hypothesis с четкими Fallback-критериями.
