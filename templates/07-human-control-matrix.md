# Human-in-the-Loop Decision Map: Ипотека за 4 часа

## ТОЧКИ ВАЛИДАЦИИ

| Точка валидации | Кто валидирует | По каким критериям | Что блокирует | Эскалация |
|-----------------|----------------|--------------------|---------------|-----------|
| **1. До запуска агента** | Product Engineer + Риск-блок | - SDD содержит Outcome Hypothesis<br>- Eval suite покрывает 80%+ кейсов<br>- Codified Guardrails включают 152-ФЗ | Запуск агента в production | CTO → Директор по рискам |
| **2. Перед merge в main** | Review Agent + Tech Lead | - Eval pass rate ≥95%<br>- Agent-introduced Regression Rate <2%<br>- Security gate пройден | Merge в main branch | Engineering Manager |
| **3. Перед deploy в staging** | QA Engineer + Security Officer | - Smoke tests passed<br>- SAST/DAST scan пройден<br>- 152-ФЗ compliance проверен | Deploy в staging | CISO |
| **4. Перед deploy в production** | Release Manager + Риск-блок | - Все evals пройдены<br>- Evidence Bundle полный<br>- Rollback plan готов | Deploy в production | CTO → Директор по рискам |
| **5. Для сложных кейсов (20%)** | Андеррайтер | - AI-агент подготовил резюме кейса<br>- Флаги риска проверены<br>- Решение обосновано | Автоматическое одобрение/отказ | Старший андеррайтер → Кредитный комитет |
| **6. Post-deploy (7 дней)** | Product Engineer + Риск-блок | - Time-to-decision <4 часов<br>- Agent-introduced Regression Rate <5%<br>- % эскалаций <25% | Масштабирование на новые сегменты | CTO → Правление |
| **7. Post-deploy (28 дней)** | Product Owner + Риск-блок | - Outcome Hypothesis подтверждена<br>- Leading indicator достигнут<br>- Fallback-критерий не сработал | Продолжение rollout | CTO → Правление |

## ПРИНЦИПЫ

### Human-in-the-loop (HITL)
- Человек участвует в каждой существенной точке решения
- Для 80% стандартных кейсов → автоматическая обработка (R2)
- Для 20% сложных кейсов → эскалация на андеррайтера (R1)

### Human-over-the-loop
- Product Engineer наблюдает за работой агента
- Еженедельный аудит одобренных заявок
- Ретроспектива при отклонении метрик

## КРИТЕРИИ ЭСКАЛАЦИИ

### Автоматическая эскалация на андеррайтера:
- Скоринг-балл <600
- Нестандартная структура дохода (ИП, самозанятые)
- Подозрение на мошенничество (флаги от службы безопасности)
- Неполный пакет документов
- Противоречия в данных (разные суммы в разных документах)

### Автоматическая эскалация на кредитный комитет:
- Сумма кредита >10 млн руб.
- Клиент в списке PEP (политическиExposed лица)
- Подозрение на отмывание денег (AML-флаги)

## МЕТРИКИ

### HITL Coverage Score
Цель: 100% SDD имеют заполненную Decision Map

### Escalation Quality Score
Цель: >85% эскалаций обоснованы (не false positive)

### Human Override Rate
Цель: 10-25% (если >40% → агент работает плохо, если <5% → человек не проверяет)
