# Проверка библиотеки шаблонов (Pattern Library)

## ПОИСК ПАТТЕРНОВ

### Найденные паттерны:
1. **Document Verification Agent** (PAT-0042)
   - Описание: AI-агент для проверки документов через OCR и валидацию
   - Статус: Published, в использовании в 3 командах
   - Eval suite: 15 тестов (точность OCR, валидация форматов)
   - Применимость: Подходит для проверки документов по ипотеке

2. **Credit Scoring Agent** (PAT-0087)
   - Описание: AI-агент для запуска скоринг-модели и интерпретации результатов
   - Статус: Published, в использовании в 5 командах
   - Eval suite: 20 тестов (точность скоринга, интерпретация)
   - Применимость: Подходит для скоринга заявок на ипотеку

3. **Underwriting Agent** (PAT-0103)
   - Описание: AI-агент для автоматического андеррайтинга стандартных кейсов
   - Статус: Published, в использовании в 2 командах
   - Eval suite: 25 тестов (точность решений, эскалация сложных кейсов)
   - Применимость: Подходит для андеррайтинга ипотеки

4. **Mortgage Application Orchestrator** (PAT-0156)
   - Описание: Оркестратор для обработки заявок на ипотеку
   - Статус: Draft (черновик)
   - Eval suite: нет
   - Применимость: Черновик, нужно доработать

### Не найдены:
- Паттерн для интеграции с БКИ/ФНС/ЕГРЮЛ через API
- Паттерн для автоматической эскалации сложных кейсов

## ПЕРЕИСПОЛЬЗОВАНИЕ

### Берем из библиотеки:
1. **Document Verification Agent** (PAT-0042)
   - Используем как есть
   - Настраиваем под документы по ипотеке (паспорт, справка 2-НДФЛ, договор ДДУ)

2. **Credit Scoring Agent** (PAT-0087)
   - Используем как есть
   - Настраиваем под скоринг-модель для ипотеки

3. **Underwriting Agent** (PAT-0103)
   - Используем как базу
   - Добавляем правила для ипотеки (проверка залоговой стоимости, ПВ)

## НОВЫЕ ПАТТЕРНЫ

### Нужно создать:
1. **Mortgage Application Orchestrator** (на базе PAT-0156)
   - Доработать черновик
   - Добавить интеграцию с БКИ/ФНС/ЕГРЮЛ
   - Добавить логику эскалации сложных кейсов
   - Написать eval suite (30+ тестов)

2. **External Data Integration Pattern** (новый)
   - Интеграция с БКИ через API
   - Интеграция с ФНС через API
   - Интеграция с ЕГРЮЛ через API
   - Обработка ошибок, retry-логика, кэширование

## ССЫЛКИ
- PAT-0042: Document Verification Agent
- PAT-0087: Credit Scoring Agent
- PAT-0103: Underwriting Agent
- PAT-0156: Mortgage Application Orchestrator (Draft)

## SPEC REUSE RATE
Ожидаемый % переиспользования: ~70%
- 3 готовых паттерна из 4 необходимых
- 1 новый паттерн (оркестратор) + 1 новый паттерн (интеграция)

Экономия токенов: ~40% (за счет использования готовых паттернов)
