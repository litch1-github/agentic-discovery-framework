markdown
# Оценка пригодности для агентного исполнения

## КРИТЕРИЙ 1: Степень неопределенности
Задача: обработка заявки на ипотеку
- Предсказуемый результат? Да (одобрить/отказать/эскалировать)
- Структурированные входные данные? Да (документы, анкеты, данные из БКИ)
- Неопределенность: СРЕДНЯЯ (стандартные кейсы предсказуемы, сложные требуют адаптации)

→ AI-NATIVE или AGENTIC

## КРИТЕРИЙ 2: Требуемая адаптивность
- Фиксированные сценарии? Нет (каждая заявка уникальна)
- Средняя гибкость? Да (агент должен адаптироваться к разным типам документов, 
  структуре дохода)
- Высокая адаптивность? Не требуется (нет открытых задач)

→ AI-NATIVE

## КРИТЕРИЙ 3: Цена ошибки
- Низкая? Нет (ошибка = одобрение рискованной заявки → финансовые потери)
- Средняя? Да (нужны проверки evals, human-in-the-loop для сложных кейсов)
- Высокая? Не критическая (есть fallback, возможность апелляции)

→ AI-NATIVE (с human-in-the-loop для сложных кейсов)

## КРИТЕРИЙ 4: Обратимость действий
- Полностью обратимые? Нет (одобренная заявка → выдача кредита → необратимо)
- Частично обратимые? Да (можно отозвать одобрение до подписания договора)
- Необратимые? Нет (клиент может отказаться)

→ AI-NATIVE (с human-in-the-loop перед финальным решением)

## КРИТЕРИЙ 5: Наличие паттерна в библиотеке
- Есть ли подходящий паттерн? Частично
  - Паттерн "Document Verification Agent" → есть
  - Паттерн "Credit Scoring Agent" → есть
  - Паттерн "Mortgage Application Orchestrator" → нет (нужно создать)

→ AI-NATIVE (с созданием нового паттерна)

## ИТОГОВОЕ РЕШЕНИЕ
Класс применимости: AI-NATIVE (SDD-driven)

Обоснование:
- Задача структурирована, но требует адаптации к разным кейсам
- Цена ошибки средняя → нужны evals и human-in-the-loop
- Действия частично обратимы → можно откатить решение
- Нужен новый паттерн для оркестрации субагентов

Уровень автономии: R2 (автономная работа в feature-branch с ревью перед merge)
- 80% заявок → автоматическая обработка (R2)
- 20% заявок → эскалация на человека (R1)

Типовой workflow: Orchestrator-Workers
- Orchestrator: координирует проверку документов, скоринг, андеррайтинг
- Workers: Document Verification Agent, Scoring Agent, Underwriting Agent
- Review Agent: проверяет решения перед merge
