# PR/FAQ: Ипотека за 4 часа

## ПРЕСС-РЕЛИЗ

### Заголовок
"Банк X запускает ипотеку за 4 часа: решение по заявке в день обращения"

### Подзаголовок
"Новый AI-агент автоматически проверяет документы, скоринг и андеррайтинг, 
сокращая время ожидания с 10 дней до 4 часов для 80% клиентов"

### Основной текст
Сегодня Банк X объявил о запуске нового сервиса обработки заявок на ипотеку 
на базе AI-агента. Теперь клиенты, подающие заявку на ипотеку в новостройках, 
получают решение в течение 4 часов вместо 7-10 дней.

AI-агент автоматически:
- Проверяет документы через компьютерное зрение (OCR + валидация)[glm-ocr-сервис распознавания]
- Запрашивает данные из БКИ, ФНС, ЕГРЮЛ через API [bki-api, fns-api, egrul-api]
- Запускает скоринг-модель и андеррайтинг параллельно [agent-scoring, agent-underwriter]
- Принимает решение для 80% заявок без участия человека [decision-engine]
- Эскалирует сложные случаи (20%) на андеррайтера с готовым резюме [human-in-loop]

 

"Мы больше не теряем клиентов на этапе ожидания. 80% заявок обрабатываются 
автоматически, а наши андеррайтеры фокусируются на сложных кейсах, где нужен 
человеческий опыт", — говорит директор розничного бизнеса.

### Цитата стейкхолдера
"Это не просто ускорение процесса. Мы перепроектировали его вокруг возможностей 
AI-агентов: параллельные проверки, автоматическая эскалация, мгновенное решение 
для стандартных случаев", — CTO Банка X.

## FAQ

**В1: Как AI-агент проверяет подлинность документов?**
О1: AI-агент использует компьютерное зрение для OCR и валидации. Все документы 
проверяются по базе ФНС и ЕГРЮЛ через API. Подозрительные случаи автоматически 
эскалируются на службу безопасности.

**В2: Что если клиент не согласен с решением AI-агента?**
О2: Клиент может подать апелляцию. Заявка автоматически передается на ручную 
проверку старшим андеррайтером. Среднее время обработки апелляции: 1 день.

**В3: Как обеспечивается безопасность персональных данных?**
О3: Все ПДН обрабатываются в изолированной среде (sandbox PROD). Данные не 
покидают контур банка. Аудит всех действий агента ведется в append-only журнале 
(соответствие 152-ФЗ, требования ЦБ).

**В4: Что если AI-агент ошибется и одобрит рискованную заявку?**
О4: AI-агент работает с уровнем автономии R2 (автономная работа в feature-branch 
с ревью перед merge). Все решения логируются. Риск-блок проводит еженедельный 
аудит одобренных заявок.

**В5: Какие 20% заявок эскалируются на человека?**
О5: Эскалация происходит при:
- Низком скоринг-балле (<600)
- Нестандартной структуре дохода (ИП, самозанятые)
- Подозрении на мошенничество (флаги от службы безопасности)
- Неполном пакете документов
